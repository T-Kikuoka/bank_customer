# 機械学習による銀行の顧客ターゲティング


## 概要
銀行の顧客データから、当該のキャンペーンの結果、定期預金口座を開設するかどうかを予測した。

---

## 製作期間
- データ確認:1週間
- 特徴量エンジニアリング:1週間
- モデル構築・結果出力:1週間

---

## 背景
2008年のリーマンショック以降、ヨーロッパの銀行は資金調達に苦しんでいた。そんな中ポルトガルのとある銀行では、安定した資金基盤を手に入れる為、定期預金を集めることを目的にしたマーケティングキャンペーンを実施した。キャンペーンは顧客に電話で定期預金を勧誘するものたったが、電話勧誘は人的コストや時間的コストがかかり、成約率も低く、やみくもに電話をかけるだけでは十分な成果が得られなかった。

この成果物では経済状況、過去のキャンペーンの実施結果などの顧客情報をもとに、その顧客が本キャンペーンで定期預金口座を開設する確率を予測し
- 成功確率の低い顧客への電話を減らし、リソースの無駄を省く
- 成功確率の低い顧客への電話を減らし、顧客が不要な電話を受け取ることの不満を無くす
- 成功確率の高い顧客への電話を増やし、営業結果を上げる

など、営業効率を向上させると同時に、顧客満足度も向上させることを目的とした。

## 工夫点
この課題において大切なのは、「口座を開設しない顧客を判別する」よりも「口座を開設する顧客を逃さない」ことだと考え、口座を開設する顧客のrecallを可能な限り上げることを目標にした。

また、LightGBMは重要な特徴量を自動的に判断して説明変数にするが、最適化されたデータ整形ができると結果が向上するため、手探りで様々な特徴量を整形して試した。

---

## 結果

||precision|recall|f1-score|support|
|-----|-----|-----|-----|-----|
|0|0.98|0.85|0.91|4791
|1|0.44|0.88|0.58|635

(0:口座を開設しない対象, 1:口座を開設する対象) 
- precision : 正解数/予測数
- recall : 正解数/support
- f1-score : precisionとrecallのバランス指標
- support : 母数

以上から、
- 「口座を開設する」と予想した対象の44%が実際に開設する
- 口座を開設する対象の88%を「口座を開設する」と予想した

ということが言え、業務効率化と機会損失の防止につながる結果となった。

---

## 効果

- 口座を開設しない対象の85%を正確に予想できる為、無駄な営業の85%を削減できる
- 口座を開設する対象の88%を正確に予想できる為、機会損失を12%に抑えることができる

仮に全対象数を10000とすると、おおよそ 0 : 8830人  1 : 1170人 となり、モデルによって顧客の分類を行った場合と行わなかった場合で以下のようになる。

||営業対象|営業成功|機会損失|
|-----|-----|-----|-----|
|未分類|10000|1170|0|
|分類済|2340|1030|140|

(※分類済とはモデルによって顧客の予想をした結果を使用したものを指す)

機会損失はあるが、分類済の方が効率よく営業できていることが分かる。また営業のリソースが限られている場合は分類済の方が機会損失が少なくなる。上記の表において、営業対象を2000で統一すると以下のようになる。

||営業対象|営業成功|機会損失|
|-----|-----|-----|-----|
|未分類|2000|334|836|
|分類済|2000|880|290|

このように限られたリソースの中では顧客を分類することによって営業利益そのものを上げることができる。
